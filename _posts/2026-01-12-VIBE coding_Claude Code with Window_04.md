### 목표

**1. 배경**

​	지금까지 배웠던 Next.js에 대한 이론적인 공부를 했는데, 직관적으로 이해가 안될 수 있다. 



**2. 목표**

​	하나하나 직접 코딩을 하며 작업을 해보자.

### 1. 실습을 위한 예제 생성

```
npx create-next-app@latest frontend
```

1. **`npx` (Node Package eXecute)**
   - **의미:** "설치하지 말고, 딱 한 번만 실행해라."
   - **설명:** 보통 프로그램은 설치(`install`)를 하고 실행하죠? 하지만 이 설치 도구는 매일 쓰는 게 아니라 프로젝트 만들 때 딱 한 번 씁니다. 그래서 컴퓨터에 영구적으로 깔지 않고, **클라우드에서 최신 버전을 잠시 빌려와서 실행하고 삭제**하는 방식입니다.
   - *비유: 드릴을 사는 게 아니라, 공구 대여소에서 잠시 빌려와서 구멍 하나 뚫고 반납하는 것과 같습니다.*
2. **`create-next-app@latest`**
   - **의미:** "Next.js 앱을 만드는 최신 버전 도구."
   - **설명:** React/Next.js 팀이 만들어둔 **공식 설치 스크립트**입니다. 이 스크립트 안에는 기본적인 폴더 구조, 필수 라이브러리 설정, 초기 파일들이 모두 담겨 있습니다. `@latest`는 구버전이 아닌 가장 최신 안정화 버전을 가져오라는 뜻입니다.
3. **`frontend`**
   - **의미:** "프로젝트 이름."
   - **설명:** 현재 위치에 `frontend`라는 이름의 폴더를 만들고, 그 안에 모든 결과물을 쏟아부으라는 목적지 설정입니다.

### 2. 작업 공간 (Work Space)

**"우리가 99%의 시간을 보내게 될 곳"**입니다. 실제 웹사이트의 화면과 기능을 만드는 곳이죠.

- **📂 `app` 폴더 (가장 중요!)**
  - 여기가 바로 **"건설 현장"**입니다.
  - `page.tsx`: (앞서 배운) 실제 화면 페이지.
  - `layout.tsx`: 웹사이트의 전체적인 틀 (헤더, 푸터 등).
  - `globals.css`: 웹사이트 전체 디자인 규칙(스타일)을 정의하는 곳.
- **📂 `public` 폴더**
  - **"자재 창고"**입니다.
  - 이미지, 폰트, 아이콘(favicon) 같은 정적 파일들을 여기에 둡니다.
  - 이 폴더에 있는 파일은 웹 브라우저가 주소를 쳐서 바로 접근할 수 있습니다. (예: `내사이트.com/image.png`)



---

### 3. 프로젝트 명세서 (Project Specs)

**"프로젝트의 설정과 규칙을 정의하는 문서"**들입니다.

- **📄 `package.json` (중요!)**
  - **"BOM (Bill of Materials, 자재 명세서)"**입니다.
  - 이 프로젝트가 어떤 라이브러리(부품)들을 쓰고 있는지, 프로젝트 이름은 뭔지, 실행 명령어(`npm run dev` 등)는 뭔지 적혀 있습니다.
- **📄 `tsconfig.json`**
  - **"안전 규격서"**입니다.
  - TypeScript가 얼마나 엄격하게 코드를 검사할지 규칙을 정해둔 파일입니다. (우리가 건드릴 일은 거의 없습니다.)
- **📄 `next.config.ts`**
  - **"엔진 튜닝 설정"**입니다.
  - Next.js 서버의 고급 설정(이미지 최적화 방식, 경로 설정 등)을 하는 곳인데, 초반에는 건드릴 일이 없습니다.

### 4. 건드리면 안 되는 곳 (Do Not Touch)

시스템이 자동으로 관리하거나, 외부에서 가져온 것들입니다.

- **📂 `node_modules`**
  - **"외부 부품 창고"**입니다.
  - `package.json`에 적힌 라이브러리들이 실제로 다운로드되어 저장되는 곳입니다. 파일이 수만 개가 들어있지만, **절대 직접 수정하면 안 됩니다.**
- **📂 `.next`** (회색으로 표시됨)
  - **"공장 가동 결과물"**입니다.
  - 우리가 코드를 짜고 실행하면, Next.js가 이걸 기계가 읽기 좋게 번역해서 여기에 임시로 쌓아둡니다. 지워도 다시 실행하면 또 생깁니다.

### 6. 첫 번째 컴포넌트 작성하기 (`page.tsx`)

이제 주인공인 페이지 파일을 작성할 차례입니다.

1. **`app/page.tsx`** 파일을 여세요.
2. **안에 있는 내용을 전부 지우세요.**
3. 아래 코드를 직접 타이핑해 보세요. (복사보다는 직접 쳐보시는 걸 추천합니다!)

TypeScript

```
// page.tsx

// 1. 컴포넌트 이름은 대문자로 시작 (Home)
export default function Home() {
  
  // 2. 화면에 보여줄 태그를 리턴 (HTML 처럼 생겼죠?)
  return (
    <div>
      <h1>안녕하세요! 첫 페이지입니다.</h1>
      <p>Next.js의 세계에 오신 것을 환영합니다.</p>
      <p>여기서부터 하나씩 만들어 봅시다.</p>
    </div>
  );
}
```

------

### 7. 코드 해설 (이게 무슨 뜻이지?)

방금 작성하신 10줄 남짓한 코드가 **React 컴포넌트의 모든 기초**를 담고 있습니다.

1. **`export default`**:
   - "이 함수를 밖으로 내보냅니다"라는 뜻입니다.
   - Next.js는 이 파일을 읽을 때 `default`로 내보내진 함수를 찾아서 화면에 그려줍니다. 이게 없으면 화면이 안 나옵니다.
2. **`function Home()`**:
   - 이것이 바로 **컴포넌트**입니다. 그냥 자바스크립트 함수일 뿐입니다.
   - **규칙:** 함수 이름(`Home`)은 반드시 **대문자**로 시작해야 합니다. (소문자로 쓰면 React가 인식을 못 합니다.)
3. **`return ( ... )`**:
   - 함수가 값을 반환하듯, 컴포넌트는 **UI(HTML 태그들)**를 반환합니다.
   - 이 문법을 **JSX**라고 부릅니다.
